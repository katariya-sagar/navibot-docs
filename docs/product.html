Product requirements

MVP scope

The first version should prove that a conversational route assistant can make route choice clearer and more confident.

Goals

  • Accept a natural-language route planning request.
  • Identify origin, destination, constraints, timing, and preferences.
  • Return a recommended route with explanation and alternatives.
  • Capture user feedback on whether the recommendation was useful.

Core requirements

  • Route intent parsing for common trip requests.
  • Route comparison based on time, reliability, cost, transfer count, and walking distance.
  • Human-readable answer with next action and trade-offs.
  • Responsive documentation and demo surface for project stakeholders.

Non-goals

  • Real-time production-grade traffic prediction in the MVP.
  • Full account system, payment flow, or fleet management.
  • Replacing a map provider. NaviBot should sit above provider data.

Success metrics

Track task completion, recommendation acceptance, user confidence, and the percentage of sessions where users understand why a route was recommended.
