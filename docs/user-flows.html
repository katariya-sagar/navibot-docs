User flows

Core journeys

The MVP should focus on a small number of flows that demonstrate decision support, not just directions.

Plan a trip

  1. User describes where they need to go and when.
  2. NaviBot extracts missing details and asks one focused follow-up question if needed.
  3. NaviBot presents the recommended route plus two alternatives.
  4. User confirms or changes a preference.

Compare routes

  1. User asks why one route is better.
  2. NaviBot compares speed, reliability, cost, transfers, and walking distance.
  3. User selects the option that best matches their constraint.

Recover from disruption

  1. Context changes because of delay, closure, or missed transfer.
  2. NaviBot identifies available alternatives.
  3. NaviBot explains the impact and gives the next action.

Feedback loop

After each recommendation, the user can mark whether the answer was useful. Feedback should be tied to route attributes, not just a generic score.
