Roadmap

Delivery plan

The roadmap keeps the project focused on the smallest useful proof of NaviBot before expanding integrations and intelligence.

Phase 1: Discovery

  • Define primary persona and trip-planning scenario.
  • Document constraints, provider assumptions, and data risks.
  • Create a test set of realistic trip prompts.

Phase 2: Prototype

  • Build a static demo of the conversational route flow.
  • Draft route cards, explanations, and comparison states.
  • Validate wording and recommendation clarity with sample users.

Phase 3: MVP service

  • Implement request parsing and route candidate normalization.
  • Add scoring rules for time, reliability, cost, and transfer complexity.
  • Capture structured feedback on recommendation usefulness.

Phase 4: Public demo

  • Publish documentation and demo through GitHub Pages.
  • Collect feedback, measure completion, and identify the highest-value integration.
  • Plan the next release around real provider data and improved context signals.
