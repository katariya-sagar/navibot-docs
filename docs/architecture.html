Architecture

System design

NaviBot uses a layered architecture so product logic stays separate from map providers, AI services, and presentation.

Layers

Client experience: chat, route cards, comparison view, feedback controls
Intent parser: extracts origin, destination, timing, constraints, and preferences
Route/context engine: gathers candidate routes, live context, and constraints
Recommendation layer: scores routes and produces ranked options
Response layer: explains the route and asks follow-up questions when needed

Data flow

  1. User asks for a route or changes a constraint.
  2. Intent parser normalizes the request into structured trip parameters.
  3. Route/context engine requests candidate routes and contextual signals.
  4. Recommendation layer ranks candidates and records scoring reasons.
  5. Response layer returns the best option, alternatives, and next action.

Quality attributes

  • Explainability: every recommendation should include the top reasons.
  • Provider flexibility: route providers should be replaceable behind adapters.
  • Privacy: store only the data needed for product improvement.
  • Resilience: degrade gracefully when real-time data is unavailable.
